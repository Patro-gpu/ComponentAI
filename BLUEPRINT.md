# ComponentAI — 商业模式重塑蓝图

## 当前状态

### 产品
- **功能**: AI 驱动的 UI 组件生成器。用户选组件类型 + 框架，输入自然语言描述，生成生产级代码
- **前端**: 单页 HTML，带代码/预览双面板 (React/Vue/Svelte/HTML/Web Component)
- **后端**: FastAPI，代理 DeepSeek API，支持切换模型 (GPT/Claude)
- **部署**: 前端 GitHub Pages，后端 Railway
- **技术栈**: FastAPI + DeepSeek + HTML/JS

### 当前缺陷
- 无商业模式（纯工具，无变现路径）
- 无用户系统（无法留存、付费）
- 无差异化壁垒（任何开发者都能用同样方案复刻）
- 无定价策略
- 无分发渠道（仅一个公网链接）

## 需要 Codex 回答的核心问题

### 1. 商业模式
- ComponentAI 应该怎么赚钱？

### 2. 产品定位
- 核心竞争力应该是什么？（速度？代码质量？预览体验？多模型支持？）
- 切哪个细分市场最有利？（独立开发者？外包团队？设计转前端？低代码平台？）

### 3. 增长策略
- 获客渠道：SEO？社交媒体？GitHub 开源引流？付费广告？
- 冷启动：如何获取前 100 个用户？

### 4. 技术演进路线
- MVP → V1 → V2 的功能优先级排序
- 哪些功能是"必须有"，哪些是"锦上添花"？

### 5. 竞争分析
- 直接竞品：v0.dev, bolt.new, Lovable, Cursor Composer
